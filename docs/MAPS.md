# Custom maps in Alfa MP

Hosters can ship their own world content — custom interiors, new neighborhoods, retextures, custom vehicles, peds, weapons — and Alfa MP streams it to players seamlessly.

This page is the technical guide for map authors and hosters. End-user UX (loading screens etc.) is documented in [ARCHITECTURE.md §4](./ARCHITECTURE.md) and the launcher README.

## TL;DR

```bash
# In your AlfaMP resource:
my-resource/
├── fxmanifest.lua
├── stream/              ← all custom content goes here
│   ├── myCustomHouse.ymap
│   ├── myCustomHouse.ytyp
│   ├── myCustomHouse_lod.ydr
│   └── myTextures.ytd
├── server/              ← optional C# / Lua
└── client/              ← optional client scripts

# fxmanifest.lua
ensure my-resource
files {
    'stream/**/*'
}
```

That's it. AlfaServer auto-detects `stream/` and streams the assets to connected players.

## Supported formats

The GTA V engine (RAGE) defines a unified set of formats. Alfa MP inherits full support from the CitizenFX foundation:

| Ext | Purpose | Generated by |
|-----|---------|--------------|
| `.ymap` | World object placement (IPL replacement) | CodeWalker, OpenIV |
| `.ymt` | Meta files (zones, scenarios, etc.) | OpenIV, Map Editor |
| `.ymf` | Manifest fragments | CodeWalker |
| `.ytyp` | Archetype definitions (typed object catalog) | CodeWalker |
| `.ydr` | Single drawable (3D model) | 3ds Max / Blender + GIMS |
| `.ydd` | Drawable dictionary (multiple models) | 3ds Max / Blender + GIMS |
| `.yft` | Fragment (breakable model — vehicles, lampposts) | 3ds Max / Blender + GIMS |
| `.ytd` | Texture dictionary (DDS inside) | OpenIV, gta5-mods toolkit |
| `.ynd` | Path nodes (vehicle AI nav) | CodeWalker |
| `.ynv` | Nav mesh (ped pathing) | CodeWalker |
| `.awc` | Audio waves | OpenIV |

**Tooling recommendations:**
- **[CodeWalker](https://github.com/dexyfex/CodeWalker)** (free, open-source) — best for `.ymap`, `.ytyp`, navigation editing
- **[OpenIV](https://openiv.com)** (free) — extracting/modifying vanilla files for reference
- **3ds Max + GIMS Evo** or **Blender + Sollumz** — modeling pipeline

## Adding custom interiors (MLOs)

Multi-Level Objects are the standard way to ship complex interiors (custom houses, restaurants, police stations). An MLO is a composite of:

- A shell `.ydr` (the building geometry)
- A `.ymap` placing the interior somewhere in the world
- A `.ytyp` defining the interior's archetype and rooms

Recommended layout:

```
stream/
├── interiors/
│   ├── mlo_my_house.ymap
│   ├── mlo_my_house.ytyp
│   ├── mlo_my_house.ydr
│   ├── mlo_my_house+hi.ydr      ← high-LOD variant
│   └── mlo_my_house.ytd
```

Naming convention: `mlo_<name>` so they sort together and avoid collisions with vanilla.

## Adding custom vehicles

Hosters can ship custom car models. Each vehicle needs:

```
stream/
├── vehicles/
│   ├── mycar.yft        ← geometry (fragment)
│   ├── mycar.ytd        ← textures
│   ├── mycar_hi.yft     ← high-LOD
│   └── mycar.ymt        ← handling, AI config
```

And declared in `vehicles.meta` + `carcols.meta` + `handling.meta` (sample templates ship under `compat-rage/examples/`).

## Performance guidance

Players have varied hardware. Follow these to keep them comfortable:

| Budget | Soft cap | Hard cap (will crash 8 GB GPUs) |
|--------|----------|--------------------------------|
| Total VRAM | 4 GB | 6 GB |
| Total RAM | 6 GB | 10 GB |
| Resource pack download | 500 MB | 2 GB |
| Single .ytd | 64 MB | 128 MB |
| Single .ydr/.yft | 32 MB | 96 MB |

Use `alfa-map-check` (CLI, ships with AlfaServer) to validate before deploy:

```bash
alfa-map-check /opt/alfaserver/server-data/resources/my-resource
```

Reports:
- VRAM budget estimate
- Broken references (`.ymap` pointing at missing `.ytyp`)
- Duplicate archetype names across resources
- Missing collisions
- Files exceeding hard caps

## Streaming UX (what players see)

```
1. Player clicks Connect
2. Launcher hands off to AlfaMP.exe with server endpoint
3. AlfaMP.exe handshakes, server sends resource manifest with sha256s
4. AlfaMP.exe checks local cache (%LocalAppData%\AlfaMP\cache\<server>\<sha>)
5. For each resource not cached, downloads from:
   • hoster's CDN URL (declared in fxmanifest.lua → preferred)
   • else from AlfaServer's built-in download endpoint (fallback)
6. Progress shown on loading screen: % files complete + MB/s + ETA
7. Once spawn-radius assets ready, player enters world
8. Distant chunks continue downloading in background
```

## Adaptive streaming (planned M5.5)

For maps >500 MB, we split download into chunks by distance from spawn:

- **Tier 0** (spawn area, ≤500 m): blocks entry, downloaded before connect
- **Tier 1** (1 km radius): background download with high priority
- **Tier 2** (full map): background download with low priority

If a player travels into Tier 2 territory before it's cached, a short ~3 s loading screen covers the chunk fetch.

## Hot-reload (dev mode)

For map authors iterating locally, enable in `server.cfg`:

```
sv_devMode true
sv_devMode_streamReload true
```

Now when you edit a `.ymap` file on disk, the server broadcasts `refresh` to connected clients and they swap the streaming entry without disconnecting. Iteration time goes from "kick everyone, restart server, reconnect" (~60 s) to "save file" (~2 s).

## Migrating maps from RAGE MP

If your RAGE MP server already ships custom maps, the formats are identical (both use the same RAGE engine). Use `rage2alfa`:

```bash
python tools/rage2alfa.py /path/to/my-rage-map-resource
```

This will:
1. Copy all `.ymap` / `.ytyp` / `.ydr` etc. into `stream/`
2. Read `_resource.toml` → emit `fxmanifest.lua`
3. Parse `client_packages/*.js` for map-loader patterns → autoload via manifest
4. Drop a `MIGRATION-NOTES.md` next to anything that needs manual review (e.g. custom IPL loaders that have no direct equivalent)

For ~90% of community RAGE maps, the conversion is fully automatic.

## Asset CDN (recommended for >100 MB packs)

By default, AlfaServer serves stream assets directly from its host. This is fine for small packs but burns hoster bandwidth at scale. Better: put assets on a CDN.

In `fxmanifest.lua`:

```lua
resource_manifest_version 'fxv2'
resource_cdn_url 'https://cdn.my-server.com/resources/my-resource/'
```

Each file in `stream/` is then fetched from `https://cdn.my-server.com/resources/my-resource/<filename>` instead of from the AlfaServer host. Cloudflare R2 + a workers script costs ~$5/month per several hundred GB of bandwidth, vs. burning your VPS's allowance.

Alfa MP will offer a **free shared CDN** for smaller community servers at public beta — sign up via the hoster dashboard.

## Restricted content

- ❌ Cannot ship original Rockstar/Take-Two assets (DLC content, leaked files) — copyright infringement
- ❌ Cannot ship assets ripped from other commercial games — same
- ✅ Can ship original content you created
- ✅ Can ship community assets with proper attribution and creator's permission
- ✅ Can ship publicly-released free mods (CodeWalker / GTA5-Mods.com / etc.) — verify each mod's license individually
